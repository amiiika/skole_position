There are five different types of position propetys in HTML. It specifies the type of positioning method used for an element:

  • static
  • fixed
  • sticky
  • absolute
  • relative

    • position:static;

    HTML elements are positioned static by default. Static positioned elements are not affected by the top, bottom, left, and right properties.An element with position: static; is not positioned in any special way; it is always positioned according to the normal flow of the page:

    An element with position: fixed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element.




    position:fixed;

    Elements that have position: fixed are immovable and hold the position specified by top: bottom: left: right or a combination of these. You can use the combinations to specify the size of the element,but if you set all the attributes (top, bottom, left, right) then you can conflict with values ​​you have set on other attributes, such as width and height. These elements will not scroll with the other elements, and it is a simple attribute to understand because the attribute is unaffected and overrides most other attributes. You use this attribute when you want to fix an attribute in a specific place in the document.



    position:sticky;

    Elements that have position: sticky actually switches between position: relative and position: fixed. The element will be relative to their starting position, they will move to one of the attributes top: bottom: left: rigth: meet. Then it will get "sticky" and get stuck. This is quite common to use on navigation elements / hub bar.



    position:absolute;

    Elements that have postion: absolute are positioned in relation to their nearest positioned ancestor, according to the rules of inheritance. If there are no positioned elements in the inheritance order, then it relates to < body>'s position and follows the scrolling.



    position:relative;

    Elemennts that have position: relative Gets its position set in relation to another element. This means that if an element, a div for example, is position: fixed, and another element is set inside that div, then the element that is relatively will be in the upper left corner of the element that is fixed if position is set to top : 0, left: 0 because it means the element is offset 0 pixels from the top, and 0 pixels from the left. This is commonly used on curtain menus, or drop-down menus as they are called in English.
